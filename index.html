Feature Flag Demo

This demo shows how Optimizely Feature Flags work from the customer's perspective. Connect your Optimizely project to see flags in action.

Setup in Optimizely (app.optimizely.com)

  1. Create a Feature Experimentation project
  2. Create these 5 feature flags:
Flag KeyUse CaseType
dark_modeToggle dark themeRollout (on/off)
promo_bannerShow/hide bannerRollout (on/off)
checkout_buttonA/B test button styleExperiment (2 variations)
pricing_displayA/B test pricing modelExperiment with variables
vip_sectionPremium-only contentTargeted delivery
  1. For pricing_display, add a variable: price_text (string)
  2. Copy your SDK Key from Settings → Environments

A/B Experiment — checkout_button
See the SDK code powering this 
// How the checkout button experiment works
const user = client.createUserContext('user-123', { country: 'US' });
const decision = user.decide('checkout_button');

if (decision.variationKey === 'variation_b') {
  button.text = 'Buy Now — Free Shipping!';
  button.style = 'green';
} else {
  button.text = 'Add to Cart';
  button.style = 'grey';
}
// Same user ALWAYS gets same variation (MurmurHash bucketing)

// Targeted: only "premium" plan users see this section
const user = client.createUserContext('user-123', {
  plan: 'premium',   // ← this attribute determines visibility
  country: 'US'
});

const decision = user.decide('vip_section');

if (decision.enabled) {
  vipSection.show();  // User has plan=premium → show VIP perks
}
// Change plan to "free" → this section disappears
